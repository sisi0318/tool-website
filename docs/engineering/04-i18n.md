# 国际化 (i18n)

## 架构

```mermaid
flowchart LR
    TS[translations.ts<br/>zh/en 完整翻译对象] --> PROV[I18nProvider]
    PROV --> CTX[I18nContext]
    CTX --> HOOK[useTranslations(namespace)]
    HOOK --> CMP[组件消费 t('key')]
    
    PROV --> |setLocale| LS[localStorage<br/>语言偏好]
```

## 实现细节

### 翻译数据结构
- 集中存储在 `lib/translations.ts`（大型单文件）
- 两层嵌套：`locale > namespace > key`
```typescript
{ zh: { common: { search: '搜索' }, hash: { title: '哈希计算器' } } }
```

### useTranslations hook
```typescript
function useTranslations(namespace: string) {
  const { locale } = useI18n()
  const t = (key: string) => translations[locale][namespace]?.[key] ?? key
  return { t, locale }
}
```
- 命名空间粒度对应每个工具，如 `'hash'`, `'json'`, `'qrcode'`
- 通用翻译在 `'common'` 命名空间

### 语言切换
- 通过 `header.tsx` 中的 `LanguageSwitcher` 组件
- 切换后通过 `setLocale` 更新 Context + 持久化到 localStorage

## 设计原则

- 翻译文件作为唯一数据源
- 组件的文本渲染 100% 通过 `t()` 函数
- 不支持动态语言扩展（运行时无法添加新语言）
- 备选策略：未找到 key 时回退显示 key 本身
